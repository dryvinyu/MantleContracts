## ADDED Requirements
### Requirement: Dark Professional Theme
The UI SHALL use a dark theme with high-contrast surfaces and a blue-forward accent palette across all screens.

#### Scenario: Default page load
- **WHEN** a user opens any primary page
- **THEN** the background is dark, text is high-contrast, and accents are blue

### Requirement: Futuristic Visual Accents
The UI SHALL include subtle gradients, glows, or geometric patterns to convey a futuristic tone without reducing legibility.

#### Scenario: Dashboard overview
- **WHEN** the dashboard renders
- **THEN** accent treatments are visible but do not obscure data or labels

### Requirement: Consistent Component Styling
Shared UI components (cards, tables, buttons, badges, modals) SHALL follow a unified visual system.

#### Scenario: Component reuse
- **WHEN** a card or table is used on different pages
- **THEN** the component appearance is visually consistent

### Requirement: Responsive Layout Integrity
Layouts SHALL remain legible and usable on mobile and desktop breakpoints.

#### Scenario: Mobile viewport
- **WHEN** a page is viewed on a mobile-sized screen
- **THEN** spacing, typography, and charts remain readable without overflow
